# Asset Scan 引擎：近期与远期目标分析

## 近期目标

当前最合理的近期目标是先**固化测试基线**，再推进真实探测能力：

1. 先补齐测试基线：覆盖 bridge 输入输出契约、指纹打分阈值、分类与 findings 生成、探针特征提取。
2. 将 Step1~3 从 mock 升级为最小真实链路：DNS 解析、优先端口探测、基础协议识别（http/https/ws）。
3. 收敛运行时可靠性：移除调试打印，改为结构化日志；明确探针失败与引擎失败边界。
4. 完善后端集成语义：统一 details 映射、空值策略、status/risk_level/summary 生成规则。
5. 最后再扩规则和样本：在真实链路与测试稳定后扩容，避免误报/漏报放大。


## 远期目标（平台化演进方向）

`asset-scan` 的远期目标是从“最小可联调原型”演进为“平台级资产发现与风险画像核心引擎”：

1. 能力目标：形成分层资产发现能力（DNS/端口/协议/服务）与多层级指纹识别（P0/P1/P2）。
2. 平台目标：保持后端编排、引擎执行、shared 契约统一的解耦边界，支持引擎独立迭代与发布。
3. 结果目标：提供可审计、可追溯、可关联的标准化输出（证据链、置信度、风险分级、误报控制）。
4. 工程目标：建立可运维执行链路（异步调度、重试/超时治理、日志审计、样本回归）。
5. 产品目标：沉淀资产台账、暴露面趋势与风险处置闭环，支撑长期治理。


## 三人分工建议（对应近期+远期目标）

| 人员 | 近期分工（先打基础） | 远期分工（平台化演进） |
| --- | --- | --- |
| 成员 A（引擎内核） | 负责 Step1~3 真实化（DNS/端口/协议）、探针执行链路与基础性能优化 | 负责多层级指纹体系（P0/P1/P2）、规则版本治理、误报漏报优化机制 |
| 成员 B（平台集成） | 负责 backend-bridge-engine 契约稳定、details 映射统一、状态与风险语义收敛 | 负责异步调度、超时/重试治理、引擎接入标准化与可运维链路建设 |
| 成员 C（质量与交付） | 负责测试基线（单测/集成）、样本与回归集、日志与文档对齐 | 负责质量看板、回归流水线、资产台账与风险趋势展示需求对接 |

建议协作节奏：A 先交最小真实探测能力，B 同步收敛平台接口，C 全程用测试与样本兜底，形成“能力开发-平台接入-质量固化”的闭环。
